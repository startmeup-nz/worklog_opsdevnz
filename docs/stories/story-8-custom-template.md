# Story 8: Custom Entry Template

**As a** developer who uses a custom worklog format different from the
built-in section headers,

**I want to** point `worklog.toml` at a Markdown template file with
`{{DATE}}` and `{{TITLE}}` placeholders,

**so that** I can define my own worklog structure without being limited
to the default "Focus / Completed / Notes / Related / Next" sections.

---

## Acceptance Criteria

### AC-8.1: Template configuration

- [ ] Setting `template = "my-template.md"` in `worklog.toml` makes the tool
      use that file for the worklog body instead of the built-in
      `[[sections]]` headers.
- [ ] The template path is resolved relative to the directory containing
      `worklog.toml` (same as `worklog_dir` resolution).

### AC-8.2: Placeholder substitution

- [ ] `{{DATE}}` in the template is replaced with the entry date in
      ISO format (`2026-05-26`).
- [ ] `{{TITLE}}` in the template is replaced with `"Work Log - {date}"`.
- [ ] Placeholders are case-sensitive — `{{date}}` is not substituted.

### AC-8.3: Template file handling

- [ ] If the template file does not exist, the tool prints an error to
      stderr and exits with a non-zero code.
- [ ] If no `template` field is set, the tool uses the built-in
      sections-based body (unchanged behaviour).

### AC-8.4: Frontmatter unaffected

- [ ] The YAML frontmatter (title, date, author, tags, draft) is
      always generated by the tool, regardless of template use.
      Templates only replace the body.

---

## Example

`worklog.toml`:
```toml
template = "my-worklog-template.md"
```

`my-worklog-template.md`:
```markdown
# {{TITLE}}

**Date:** {{DATE}}

Write whatever you want here — no predefined sections.
```
